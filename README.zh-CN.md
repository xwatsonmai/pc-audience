# PC Audience 中文文档

PC Audience 是一个桌面端 AI 弹幕观众应用。它会定期观察你的电脑屏幕，让视觉模型总结当前行为，再让文本模型模拟直播间观众，生成从屏幕上方飘过的弹幕。

这个项目当前还是 MVP，目标是先把“自己用起来”的体验跑通：能观察、能记忆行为轨迹、能生成弹幕、能保护隐私、能调试请求。

## 核心功能

- 定期截取屏幕帧，默认每 10 秒观察一轮。
- 最近 6 帧只保存在内存中，并拼成一张 contact sheet 发给视觉模型。
- 视觉模型输出结构化观察：当前行为、应用名、摘要、置信度、是否敏感、与今日任务关系。
- 后台把多轮观察合并成连续行为片段，例如“写代码 12 分钟 -> 查资料 8 分钟 -> 看视频 6 分钟”。
- 文本模型根据当前观察、行为轨迹、最近弹幕和今日任务生成弹幕。
- 弹幕以透明置顶 overlay 的形式显示，默认点击穿透。
- 支持 OpenAI-compatible 和 Ollama，两类模型可分别用于视觉分析和弹幕生成。
- 提供 AI Debug 区，可以查看请求参数、返回内容和图像预览。

## 隐私策略

默认隐私原则是：不保存截图，只保存摘要。

- 原始帧和 contact sheet 只存在内存里。
- 不建立截图或录屏持久表。
- SQLite 只保存结构化观察、行为片段、滚动总结、诊断日志和已生成弹幕。
- 可以配置敏感应用跳过观察，例如密码管理器、银行、钥匙串等。
- 支持“捕获时隐藏弹幕”，避免模型看到自己生成的 overlay。
- 视觉模型判断画面可能敏感时，会跳过弹幕生成。

本地数据库位置在 Electron `userData` 目录中，文件名为 `pc-audience.sqlite`。

## 模型接入

应用把“模型提供方”和“模型用途”分开设计。

你可以分别配置：

- 视觉分析模型
- 弹幕生成模型

每个用途都可以选择：

- OpenAI-compatible
- Ollama

这意味着可以组合使用，例如：

- Ollama 跑本地视觉模型，OpenAI-compatible 生成弹幕。
- OpenAI-compatible 做视觉分析，Ollama 本地生成弹幕。
- 两者都使用 Ollama。
- 两者都使用 OpenAI-compatible。

### OpenAI-compatible

需要填写：

- Base URL，例如 `https://api.openai.com/v1`
- API Key
- 视觉模型名
- 文本模型名

请求会带上关闭思考相关字段：

```json
{
  "think": false,
  "thinking": false,
  "enable_thinking": false
}
```

如果端点不支持这些字段，应用会直接报错并展示在 Debug 中，不会自动删除这些字段重试。

### Ollama

默认地址：

```text
http://localhost:11434
```

需要填写：

- Ollama 视觉模型名
- Ollama 文本模型名

Ollama 使用自己的 `/api/chat` 协议，弹幕生成使用流式返回。

## 弹幕生成逻辑

当前弹幕生成分为三层。

### 1. 行为观察

视觉模型只负责客观描述，不负责吐槽。

它会输出类似：

```json
{
  "activityLabel": "research",
  "appName": "Chrome",
  "summary": "用户正在查看技术文档",
  "confidence": 0.9,
  "taskRelation": "on_task",
  "isSensitive": false
}
```

### 2. 行为记忆

后台会把多轮观察合并成连续行为片段。

例如：

```text
Codex: focused_work x6 -> Chrome: research x3 -> Bilibili: video x4
```

这让弹幕可以引用上下文，而不是只针对当前一帧说话。

### 3. 直播间导演层

弹幕模型不会直接拿完整 JSON 硬生成，而是拿到一段简短的行为轨迹简报和“直播间导演指令”。

导演层会判断：

- 当前画面是否变化很少。
- 是否刚刚切换应用或行为。
- 是否偏离今日任务。
- 是否贴近今日任务。
- 最近弹幕里哪些主题说太多了。
- 哪些机器味词不要再说。

然后动态指导模型：

- 画面变化少时，减少行为评论，多闲聊和接梗。
- 刚切换时，多做“剧情变化”的反应。
- 偏离任务时，轻度调侃行为，但不攻击人。
- 贴近任务时，多陪跑、捧场、轻微吐槽。

### 4. 观众池

弹幕模型会模拟多个观众视角，但不会输出身份标签。

观众池包括：

- 工作党
- 路过观众
- 同样摸鱼的人
- 技术观众
- 生活流观众
- 捧哏观众
- 温柔陪跑
- 阴阳怪气朋友
- 未来的自己

输出仍然是一行一条纯文本，例如：

```text
这波窗口切换有点像剧情突然转场
先别急着下结论，主播像是在找资料
我这边也在装忙，大家互相体谅一下
```

## 弹幕播放逻辑

弹幕不是一次性刷一整批，而是先进入 overlay 队列，再由播放层消费。

播放层会根据：

- 当前同屏弹幕数量
- 弹幕储备数量
- 弹幕飞行进度
- 用户设置的弹幕密度
- 用户设置的弹幕速度

动态决定什么时候发下一条。

如果储备少，会等上一条飞到屏幕中间附近再发下一条。如果储备充足，会连续补充，但仍然优先在屏幕上方 lane 出现，只有拥挤时才往中下方溢出。

项目已经移除固定文案 fallback。没有 AI 弹幕时就保持安静，不再硬塞罐头句。

## 屏幕捕获规则

- 默认跟随当前鼠标所在屏幕截图。
- 鼠标会被绘制到截图里，作为白色黑边小箭头。
- 弹幕 overlay 固定出现在主屏幕。
- 捕获前可以隐藏 overlay，避免 AI 看到自己的弹幕。
- 如果当前有弹幕正在飞，应用会优先保证观感连续，避免频繁隐藏 overlay 造成闪烁。

## 常用设置

### 今日任务

今日任务是可选的。

填写后，弹幕会多一个判断角度，例如：

- 当前行为是否贴近任务。
- 是否偏离任务。
- 是否在合理休息。
- 是否从偏离状态回到任务。

不填写时，弹幕仍然会根据行为连续性和直播间氛围生成。

### 弹幕密度

表示屏幕上同时展示的弹幕数量目标。

密度越高，同时飞过的弹幕越多，但不代表每轮 AI 必须硬生成更多内容。

### 弹幕速度

控制弹幕横穿屏幕的速度。

默认偏慢，避免弹幕太快造成压迫感。

### 敏感应用

每行一个应用名片段。命中后会跳过观察和弹幕生成。

默认包括：

```text
1Password
Bitwarden
Keychain Access
Password
Bank
Authenticator
```

## AI Debug

主界面包含 AI Debug 区，可以查看：

- provider
- purpose，区分 vision 和 danmaku
- endpoint
- model
- timeout
- request body
- response body
- 图像预览

弹幕生成是流式请求，Debug 中会保留最终拼接出的完整文本和解析出的弹幕列表。

## 当前限制

- MVP 主要以 macOS 实机验证为主。
- 观察频率固定为 10 秒，暂未开放 UI 调整。
- 数据库目前只保存摘要和弹幕，没有做长期记忆压缩策略。
- 弹幕质量依赖文本模型本身，小模型可能更容易重复或输出模板句。
- 还没有打包安装器，当前以开发运行方式启动。

## 项目结构

```text
src/main
  Electron 主进程、数据库、屏幕观察、AI 调用、行为记忆

src/renderer
  React 主界面和弹幕 overlay

src/shared
  共享类型、默认设置、隐私判断、JSON 解析、安全过滤、contact sheet

tests
  行为合并、弹幕安全、JSON 解析、观察配置等测试
```

## 设计边界

弹幕可以轻微调侃行为，但不应该攻击用户本人。

允许：

- 调侃窗口切换。
- 调侃偏离任务。
- 开玩笑说“又开始奖励自己”。
- 像直播间观众一样闲聊。

不允许：

- 攻击人格、智力、外貌、身份、健康。
- 泄露或复述敏感内容。
- 把隐私画面当成弹幕素材。
- 使用固定文案假装 AI 生成。
